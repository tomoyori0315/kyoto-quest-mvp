# AGENTS.md — 京都クエスト 開発ルール

## プロジェクト概要
京都・伏見稲荷を舞台にしたスマホ縦画面(9:16)の探索型Web RPG。
固定背景＋局所自由移動型。背景は画像、主人公/NPC/猫/UIは別レイヤーでコード描画する。

## 作業スタイル
- 返答は日本語で行う
- まず結論、その後に理由と変更内容を説明する
- 大きな変更の前には、必ず変更方針を説明してから着手する
- ユーザーはコード実装の細部判断ではなく、ゲーム性・京都らしさ・見た目・操作感を判断する
- ユーザーに確認してほしいことは、チェック項目として明確に出す

## 技術方針
- 外部ライブラリなし、または最小構成で動くことを優先
- Canvas 2Dを基本とする
- スマホSafariでのタッチ操作と、PCでのキーボード操作の両方を維持する
- GitHub Pagesにそのまま置ける構成を維持する
- 既存の動作を壊さないことを最優先にする

## 背景の鉄則
- 背景画像に主人公・NPC・猫・UI・会話ウィンドウを描き込まない
- 主人公・NPC・猫・UIは背景とは別レイヤーで実装する
- 背景画像とは別に、歩ける範囲・対象配置・出口をJSONで持つ
- 背景PNGは9:16の縦構図を基本にする

## データ設計のルール
- 近接判定は「半径内で一番近い1つ」を拾う想定
- 対象同士を近づけすぎない
- 前に進む出口は奥、戻る出口は手前に分ける
- 対象の判定中心は、必ず歩ける範囲から反応半径内に届く位置に置く

## コンテンツ方針
- 神社・信仰を茶化さない
- 学びは説明文の押し付けにせず、会話や調査で自然に出す
- 詳しい説明は「もっと読む」で展開する
- 実在の神社・地名は使ってよい
- 猫は架空のゲーム内キャラとして扱う
- ご朱印はリアルなご朱印とは別物の「参拝記録」「旅の記録」「記念印」として扱う

## 変更時のルール
- 1回の作業で大きく変えすぎない
- 変更後は、ユーザーが確認すべき操作を必ず書く
- エラーや不明点があれば、推測で進めずに確認する
